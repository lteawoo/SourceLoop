## Why

SourceLoop already plans bounded research batches, but its built-in planner still starts from a fixed generic template instead of the user’s actual topic, goal, and intended output. To match the intended workflow, SourceLoop needs a first-class path where an external AI can generate topic-specific research questions and SourceLoop can validate, archive, and run that batch without falling back to generic prompts.

## What Changes

- Add a planning input path that accepts AI-authored question drafts for a topic-backed research batch.
- Validate imported question drafts against SourceLoop’s supported question families and archive contract before a run is created.
- Preserve existing planning controls such as `maxQuestions` and `families` when operators apply them to AI-authored question batches.
- Keep the built-in template planner as a fallback when no AI-authored question batch is provided.
- Update operator guidance so Codex, Claude Code, and Gemini workflows prefer topic-specific AI-authored questions when available.

## Capabilities

### New Capabilities
- `ai-authored-question-planning`: Accept, validate, archive, and execute topic-specific question drafts generated by an external AI system.

### Modified Capabilities
- `research-question-planning-controls`: Planning scope controls must continue to work when the batch originates from AI-authored questions instead of the built-in template planner.

## Impact

- Affects `plan` CLI behavior, planning-domain validation, and archived question batch metadata.
- Adds a formal contract between external AI question generation and SourceLoop planning/execution.
- Preserves existing run execution and archive flows while changing how question batches can be sourced.
